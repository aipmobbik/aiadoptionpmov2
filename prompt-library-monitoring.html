Risk Planning
What-If Impact Analysis
Model scenarios and assess impact before decisions are made
Specific Heavy context ChatGPT · Claude · Gemini
▼

Input

  • Project schedule / Gantt chart (current)
  • Resource plan with task assignments
  • Proposed change or delay scenario
💡 Describe the exact change scenario — "2-week Build delay" is more actionable than "delay".

Output

  • Impact on Go-Live date per scenario
  • Critical path recalculation
  • Mitigation options with tradeoffs

When to use

When a delay or change is proposed. Attach project schedule + resource plan.

Template
You are a [role, e.g., Senior PMO Risk Analyst]. Run a What-If analysis on [Project Name] schedule for the scenario: [describe change/delay]. Attach: [project schedule + resource plan]. Assume [X-week delay in Phase Y]. Show best-case, likely-case, worst-case outcomes. [Director-level] tone. Output: Impact summary per scenario (Go-Live date shift, affected tasks) Revised critical path Top 3 mitigation options with effort and risk tradeoffs Evaluation: Confirm the critical path recalculation is consistent with the schedule. Flag assumptions made.
Full example
You are a Senior PMO Risk Analyst with expertise in schedule risk modeling. Run a What-If analysis on ERP Integration project schedule for the scenario: delay on Phase 2 in requirement gathering. Attach: project schedule + resource plan. Assume 1-week delay. Show best-case, likely-case, worst-case outcomes. Director-level tone. Output: Impact summary per scenario (Go-Live date shift, affected tasks) Revised critical path Top 3 mitigation options with effort and risk tradeoffs Evaluation: Confirm the critical path recalculation is consistent with the schedule. Flag assumptions made.

Change Management & Change Requests
Change Impact Assessment
Assess, document, and communicate project changes with speed and clarity
Specific Heavy context ChatGPT · Claude
▼

Input

  • Change Request (CR) Register
  • Current project plan (Gantt + resource plan)
  • Relevant scope baseline section
💡 Include the project's critical path — AI identifies whether the change directly impacts delivery.

Output

  • Impact per dimension: Scope / Schedule / Budget / Resource / Risk
  • Overall change risk rating (Low / Medium / High)
  • Approval recommendation with conditions

When to use

After a CR is submitted — before it is approved, to quantify the full impact.

Template
You are a [role, e.g., Senior PMO Business Analyst]. Perform a Change Impact Assessment for [Project Name]. CR: [CR number]. Assess: scope, schedule, budget, resource, risk. Attach: [CR form + current project plan + scope baseline]. Note the critical path and any fixed go-live dates. Factual, evidence-based tone. Output: Impact summary per dimension: Scope | Schedule | Budget | Resource | Risk (H/M/L) Overall change risk rating with rationale Approval recommendation with conditions or caveats Evaluation: Confirm impact estimates are sourced from attached documents. Flag any dimension without a quantified estimate.
Full example
You are a Senior PMO Business Analyst with expertise in change impact evaluation. Perform a Change Impact Assessment on Super App project. Assess: scope, schedule, budget, resource, and risk impact. The attached CR document relates to the Thailand Post Super Application project delivered by Bluebik Group PCL. The project critical path runs sequentially through Design → Build → Test → Deploy. All impact estimates must be traceable to the attached CR or project Timeline sheet. Do not infer, assume, or estimate any figure not present in the source documents. Evidence-based tone. Output: Impact summary per dimension | Overall risk rating | Approval recommendation Evaluation: Verify all impact estimates reference the attached plan. Flag anything unquantified.

Draft Change Request (CR) Form
Assess, document, and communicate project changes with speed and clarity
Specific Light context Claude
▼

Input

  • Scope change request or email thread from stakeholder
  • Current approved project scope baseline
  • Relevant section of the project plan
💡 Paste the requestor's exact email text — avoids interpretation errors in the CR.

Output

  • Formal CR form with impact assessment
  • Schedule & budget impact summary
  • Approval recommendation & next steps

When to use

When a scope change is requested — immediately after stakeholder discussion.

Template
You are a [role, e.g., Senior PMO Change Manager]. Draft a formal Change Request form for [Project Name]. CR#: [CR number]. Change: [brief description]. Attach: [scope change email/request + current scope baseline]. Include: change description, business justification, schedule/budget/risk impact, approval recommendation. [Formal] tone. Output: CR form with all standard fields. Evaluation: Verify all CR fields are populated. Flag if budget/schedule impact is missing or unquantified.
Full example
You are a Senior PMO Change Manager with expertise in formal change governance. Draft a formal CR form for the scope change requested. Attach: the CR document. Include all standard CR fields. Budget impact in person-days. No speculation — use attached documents only. Formal tone. Output: CR form with all standard fields. Evaluation: Confirm all fields are populated. Flag any impact estimate that lacks a source.

Project Analysis & Governance
Project Health Check Assessment
Help PMOs understand project information faster and identify risks early
Specific Heavy context ChatGPT · Claude · Gemini
▼

Input

  • Project plan & milestone tracker (current)
  • RAID log (latest version)
  • Status report (latest period)
💡 Attach the RAID log with all open items — AI flags escalation-worthy risks automatically.

Output

  • RAG status per dimension (Scope/Schedule/Budget/Risk/Team)
  • Top 3 escalation-worthy issues
  • Recommended corrective actions with owners

When to use

Weekly or at phase gates. Attach project plan + RAID log + status report.

Template
You are a [role, e.g., PMO Reviewer]. Conduct a health check assessment for [Project Name] as of [date/week]. Attach: [project plan + RAID log + status report]. RAG-rate each dimension. Flag items requiring escalation. [Director-level] tone. Output: RAG dashboard: Scope | Schedule | Budget | Risk | Team (with 1-line rationale each) Top 3 issues requiring escalation Recommended corrective actions with suggested owners Evaluation: Cross-check RAG ratings against source data. Flag if any dimension contradicts the status report.
Full example
You are an experienced PMO Reviewer with expertise in enterprise project governance. Conduct a health check assessment for the ERP Integration project as of Week 11. Attach: the milestone tracker, the latest RAID log (22 open items), and the Week 11 status report. RAG-rate each dimension. Flag items requiring Steering Committee escalation. Director-level tone. Output: RAG dashboard: Scope | Schedule | Budget | Risk | Team (with 1-line rationale each) Top 3 issues requiring escalation Recommended corrective actions with suggested owners Evaluation: Verify RAG ratings are supported by the attached documents. Flag any contradictions.

Project Coordination & Follow-up
Dependency & Blocker Detection
Help PMOs focus on the most critical actions and proactively manage blockers
Specific Heavy context ChatGPT · Claude · Gemini
▼

Input

  • Project tracker / Gantt (current)
  • Backlog with task statuses
  • Latest status report
💡 Tag tasks as "blocked" or "waiting" in your tracker — AI identifies cascading impacts faster.

Output

  • Cross-team dependency map with impact assessment
  • Blocker list with resolution recommendations
  • Cascading risk analysis (what else breaks if blocked)

When to use

Before sprint planning or milestone reviews. Attach project tracker + backlog + status report.

Template
You are a [role, e.g., Senior PMO Analyst]. Review the attached tracker and backlog for [Project Name]. Identify all cross-team dependencies and active blockers. Attach: [project tracker + backlog with statuses + latest status report]. Flag all items with status "blocked" or "waiting". Show cascading impact. [PM-facing] tone. Output: Dependency map: Task | Depends On | Team | Status | Impact if Delayed Blocker list with resolution recommendation and owner Cascade analysis: top 3 items that will break if blockers persist Evaluation: Verify all blockers reference a specific task ID. Flag blockers with no resolution plan or owner.
Full example
You are a Senior PMO Analyst for the Super App project. Identify all cross-team dependencies and active blockers in the Super App project. Attach: the current project tracker, backlog (47 tasks), and the current status report. Flag all "blocked" / "waiting" tasks. Show cascading impact. PM-facing, concise tone. Output: Dependency map: Task | Depends On | Team | Status | Impact if Delayed Blocker list with resolution recommendation and owner Cascade analysis: top 3 items that will break if blockers persist Evaluation: Verify all blockers reference a specific task ID. Flag blockers with no resolution plan or owner.

Reporting & Executive Communication
Steering Committee Deck Content
Transform project information into management-ready reports
Specific Heavy context Claude
▼

Input

  • Project updates (scope, schedule, budget, risks)
  • Previous SteerCo minutes / action tracker
  • Steering Committee slide template (if available)
💡 Attach previous minutes — AI checks open actions and shows progress automatically.

Output

  • SteerCo slide content per section (Status, Risks, Decisions)
  • Action item follow-up table (from previous meeting)
  • Decision papers with recommendation and options

When to use

Before a SteerCo meeting. Attach latest project updates + SteerCo template.

Template
You are a [role, e.g., PMO Consultant preparing a SteerCo deck]. Generate Steering Committee slide content for [Project Name] — meeting on [date] using the attached BBIK PowerPoint template as the primary design and formatting reference. Attach: [project updates + previous SteerCo minutes + slide template (optional)]. Executive tone. Each slide max 5 bullet points. Flag items needing Board decision. [Director-level] language. Output: Status slide | Risks & Issues slide | Decisions slide | Action follow-up slide Evaluation: Verify all decision items have a clear recommendation. Flag any slide with more than 5 bullets.
Full example
You are a PMO Consultant preparing the Super App SteerCo deck for the Board meeting (Week 11). Generate slide content for the Super App Steering Committee — Week 11 meeting using the attached SteerCo slide template as the primary design and formatting reference. Attach: the Week 11 project updates and the previous SteerCo slide template. Executive tone. Max 5 bullets per slide. Flag Board decision items. Director-level language. Output: Status slide | Risks & Issues slide | Decisions slide | Action follow-up slide Evaluation: All decision items must have a clear recommendation. Flag slides exceeding 5 bullets.

Executive Summary Generation
Transform project information into management-ready reports
Specific Light context ChatGPT · Claude · Gemini
▼

Input

  • Status report (current period)
  • RAID log (latest version)
  • Project tracker / Gantt (current)
💡 Always use the current period's status report — outdated data produces misleading summaries.

Output

  • One-page executive summary with RAG status
  • Key decisions required from leadership
  • Upcoming milestones & critical risks

When to use

Before steering committee or executive updates. Attach status report + RAID + tracker.

Template
You are a [role, e.g., PMO Consultant]. Generate a one-page executive summary for [Project Name] for [audience] as of [date/week]. Attach: [status report + RAID log + project tracker]. Use formal, concise language. No jargon. Highlight decisions needed. [Director-level] tone. Output: Overall RAG status with 1-sentence justification 3 key accomplishments this period 2–3 issues/risks requiring executive attention Decisions required (numbered) Upcoming milestones — next 2 weeks Evaluation: Every item must reference a source document. Flag anything stated without evidence.
Full example
You are a PMO Consultant with 10+ years of executive reporting experience. Generate a one-page executive summary for the Digital Payments Platform project for the Steering Committee. Attach: the current status report, RAID log, and project tracker. Formal, concise language. No jargon. Highlight decisions needed. Director-level tone. Output: Overall RAG status with 1-sentence justification 3 key accomplishments this period 2–3 issues/risks requiring executive attention Decisions required (numbered) Upcoming milestones — next 2 weeks Evaluation: Every item must reference a source document. Flag anything stated without evidence.
